# 🔴 PART 1: OTP SYSTEM (SECURE IMPLEMENTATION)

## 🧩 DATABASE

Create table: otp_codes

Fields:

* id
* user_id (nullable)
* phone_number
* otp_hash (NOT plain OTP)
* expires_at
* attempts
* created_at

---

## 🔐 OTP GENERATION

* Generate 6-digit OTP
* Hash OTP using SHA-256 before storing
* NEVER store or log plain OTP

---

## 📲 SEND OTP ENDPOINT

POST /api/otp/send

Logic:

* Validate phone format (use libphonenumber)
* Rate limit: max 3 requests per hour per phone/IP
* Generate OTP
* Hash OTP
* Store in DB with 5-minute expiry
* Send SMS via configured provider

---

## 🔒 VERIFY OTP ENDPOINT

POST /api/otp/verify

Logic:

* Fetch latest OTP record
* Check expiry
* Max attempts = 5
* Compare hash
* If valid:

  * mark user.phone_verified = true
  * delete OTP record

---

## 🟠 SECURITY RULES

* DO NOT log OTP anywhere
* DO NOT return OTP in API responses
* Add rate limiting to send + verify
* Prevent brute force attempts
